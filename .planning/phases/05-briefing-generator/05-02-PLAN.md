      import asyncio, logging, ollama
      from datetime import datetime, timezone, timedelta
      from config import OLLAMA_MODEL, OLLAMA_URL, OLLAMA_TIMEOUT
      from opencti_client import build_pycti_client

    SECTOR_KEYWORDS set: {"finance", "critical-infrastructure", "healthcare", "energy", "government"}

    Functions to implement (in order — later functions depend on earlier ones):
      1. _make_updated_at_filter(period_hours: int) -> dict
         since = (datetime.now(timezone.utc) - timedelta(hours=period_hours)).isoformat()
         return {"mode": "and", "filters": [{"key": "updated_at", "values": [since]}], "filterGroups": []}

      2. _extract_sectors(indicators: list[dict]) -> list[str]
         Iterate indicator objectLabel lists; collect label values matching SECTOR_KEYWORDS; return sorted list.

      3. _safe_list(entity, **kwargs) -> list — inner helper inside _collect_threat_data (or module-level).
         try: return entity.list(**kwargs) or []
         except Exception as exc: log warning with exc; try: return entity.list(first=10) or []; except: return []
         (assumption A2: not all entity types may accept filters kwarg)

      4. _collect_threat_data(client, period_hours: int) -> dict
         Build filter via _make_updated_at_filter(period_hours).
         Call _safe_list for each entity type with filters=filters, first=10.
         For indicators: also pass orderBy="updated_at", orderMode="desc".
         D-04: sort indicators by x_opencti_score descending, take first 10.
         Return {"indicators": ..., "actors": ..., "malware": ..., "campaigns": ..., "attack_patterns": ..., "sectors": _extract_sectors(indicators)}

      5. _build_stats_block(data: dict, period_hours: int) -> str
         Implement per RESEARCH.md lines 418–447. Use .get() for all field access (Pitfall 6). Format:
           "Period: last {period_hours}h (ending {now}).\n"
           "New IOCs: {count} ({type breakdown}), top confidence: {score:.2f}.\n"
           "Active threat actors: {names}.\n"
           "Active malware: {names}.\n"
           "Active campaigns: {count} identified.\n"
           "Top ATT&CK techniques: {id (name), ...}.\n"
           "Affected sectors: {sectors}.\n"
         If iocs list is empty, omit confidence line and use "none" for each relevant field.

      6. _call_ollama(stats_block: str) -> str
         response = _ollama_client.chat(model=OLLAMA_MODEL, messages=[system+user], options={"temperature": 0.3})
         text = response.message.content.strip()
         words = text.split(); if len(words) > 320: text = " ".join(words[:300]) + "..."
         return text

      7. _run_generate_sync(briefing_id: str, period_hours: int) -> None
         client = build_pycti_client(); data = _collect_threat_data(client, period_hours)
         stats_block = _build_stats_block(data, period_hours); text = _call_ollama(stats_block)
         briefings[briefing_id]["text"] = text; briefings[briefing_id]["status"] = "done"

      8. async def run_generate(briefing_id: str, period_hours: int) -> None
         try: await asyncio.to_thread(_run_generate_sync, briefing_id, period_hours)
         except Exception as exc:
           logger.error("[generator] generation failed: %s", exc)
           briefings[briefing_id]["status"] = "error"; briefings[briefing_id]["error"] = str(exc)
  </action>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| environment → config.py | OPENCTI_TOKEN, OLLAMA_URL read from env; never echoed to logs |
| briefing-generator → OpenCTI | pycti reads over internal Docker network (http://opencti:8080); no auth bypass possible from this service |
| briefing-generator → Ollama | ollama.Client calls internal Docker service; OLLAMA_URL is env-configured (not user-supplied at request time) |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-05-02-01 | Information Disclosure | config.py OPENCTI_TOKEN logging | high | mitigate | logger.info logs bool(OPENCTI_TOKEN) only — never the token string. ASVS V14.2. |
| T-05-02-02 | Tampering | OLLAMA_URL env var | medium | accept | OLLAMA_URL is set at deploy time in docker-compose.yml; not user-controllable per request. Internal Docker network only. SSRF surface is limited to the compose network. |
| T-05-02-03 | DoS | _collect_threat_data pycti calls | low | mitigate | first=10 cap on all list calls (D-02). _safe_list fallback prevents crash on pycti failure. |
| T-05-02-04 | Tampering | LLM output injection via OpenCTI entity names | low | accept | Stats block assembles counts and entity names from OpenCTI (controlled data). Output goes to PDF text only — no shell execution, no HTML rendering. Risk is low. |
| T-05-02-05 | DoS | Ollama timeout — llama3.2:3b prose takes 30-45s | medium | mitigate | OLLAMA_TIMEOUT=60 default in config.py; passed to ollama.Client(timeout=OLLAMA_TIMEOUT). Generation runs in asyncio.to_thread — event loop not blocked. |
