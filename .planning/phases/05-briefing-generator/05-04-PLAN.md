    Imports:
      import logging, uuid
      from datetime import datetime, timezone
      from fastapi import BackgroundTasks, FastAPI, HTTPException
      from fastapi.responses import Response
      from pydantic import BaseModel, Field
      from generator import briefings, run_generate

    Setup:
      logging.basicConfig(level=logging.INFO)
      logger = logging.getLogger(__name__)
      app = FastAPI(title="briefing-generator", version="1.0.0")

    Model:
      class GenerateRequest(BaseModel):
          period_hours: int = Field(default=24, ge=1, le=720)

    POST /generate:
      @app.post("/generate")
      async def generate(body: GenerateRequest, background_tasks: BackgroundTasks):
          briefing_id = str(uuid.uuid4())
          # ponytail: init BEFORE add_task — task reads briefings[briefing_id] before this line otherwise
          briefings[briefing_id] = {
              "status": "generating",
              "text": None,
              "created_at": datetime.now(timezone.utc).isoformat(),
              "period_hours": body.period_hours,
              "error": None,
          }
          background_tasks.add_task(run_generate, briefing_id, body.period_hours)
          return {"briefing_id": briefing_id, "status": "generating"}

    GET /briefings/{briefing_id}:
      @app.get("/briefings/{briefing_id}")
      async def get_briefing(briefing_id: str):
          if briefing_id not in briefings:
              raise HTTPException(status_code=404, detail="briefing not found")
          return {"briefing_id": briefing_id, **briefings[briefing_id]}

    GET /briefings/{briefing_id}/pdf:
      @app.get("/briefings/{briefing_id}/pdf")
      async def get_briefing_pdf(briefing_id: str):
          if briefing_id not in briefings:
              raise HTTPException(status_code=404, detail="briefing not found")
          entry = briefings[briefing_id]
          if entry["status"] != "done":
              raise HTTPException(status_code=404, detail="briefing not ready")
          from pdf_renderer import render_pdf          # lazy import — pdf_renderer only needed on this path
          pdf_bytes = render_pdf(entry)
          return Response(content=pdf_bytes, media_type="application/pdf")

    GET /briefings:
      @app.get("/briefings")
      async def list_briefings():
          return [
              {"briefing_id": bid, "created_at": v["created_at"],
               "period_hours": v["period_hours"], "status": v["status"]}
              for bid, v in briefings.items()
          ]

    GET /health:
      @app.get("/health")
      async def health():
          return {"status": "ok"}

    NOTE: No lifespan block needed — BackgroundTasks is per-request (not startup). Intel-extractor's pattern is correct here; semantic-engine's lifespan is for startup indexing loop which this service does not need.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP client → POST /generate | period_hours is user-supplied; validated by Pydantic before any processing |
| HTTP client → GET /briefings/{id} | briefing_id is user-supplied path param; looked up in dict (no filesystem access) |
| HTTP client → GET /briefings/{id}/pdf | status checked before render_pdf called; render_pdf never receives None text |
| briefing-generator → Phase 6 dashboard | Dashboard at localhost:3000 calls these endpoints; no auth required (internal demo scope) |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-05-04-01 | Tampering | period_hours input | medium | mitigate | Pydantic Field(ge=1, le=720) rejects 0, negative, and >720 values with HTTP 422 before any pycti or ollama call. ASVS V5.1. |
| T-05-04-02 | Tampering | briefing_id path parameter — non-UUID string | low | mitigate | Dict key lookup: if briefing_id not in briefings → 404. No filesystem access, no SQL, no eval. Non-UUID strings simply miss the dict and return 404. |
| T-05-04-03 | DoS | Concurrent POST /generate requests | medium | accept | No concurrency limit at demo scope. Each request spawns one BackgroundTask (one thread via asyncio.to_thread). Ollama is single-model; concurrent LLM calls queue in Ollama. Add semaphore in v2 if throughput matters. ponytail: global serial LLM calls, add per-request semaphore if concurrent briefing DoS observed |
| T-05-04-04 | DoS | GET /briefings/{id}/pdf — render_pdf on large text | low | accept | LLM output truncated to 320 words in _call_ollama before storage. render_pdf input is bounded. |
| T-05-04-05 | Information Disclosure | GET /briefings/{id} — briefing text exposure | low | accept | Service is internal Docker network (port 8003 not exposed beyond localhost in production compose). No auth at demo scope — acceptable per STATE.md architecture constraints. |
| T-05-04-SC | Tampering | npm/pip/cargo installs | high | mitigate | All packages pre-approved in RESEARCH.md Package Legitimacy Audit; pinned in requirements.txt; built in Docker layer with --no-cache-dir. No new installs in this plan. |
</threat_model>
