## Layout

### Navigation
- Horizontal tab bar fixed at top, full width.
- Three tabs: "Overview" | "Threat Hunt" | "Briefings".
- Active tab: accent underline (3px, accent #3b82f6), text weight 600.
- Inactive tab: text muted, weight 400.
- Tab bar background: secondary #1a1d27.
- No sidebar. No mobile breakpoints — REQUIREMENTS.md Out of Scope: "Mobile/responsive dashboard".

### Overview View (DASH-01, DASH-02)
- Two rows of cards.
- Row 1: Feed health grid — one card per feed. Each card shows: feed name (label/12px/600), last_run timestamp (body/14px/400, muted), IOC count (display/28px/600), status dot (8px circle, success/warning/destructive color).
- Row 2: Summary stats bar — "IOC count (24h)" stat card left, "Top ATT&CK Techniques" list card right.
- Top techniques list: ordered list, max 5 items, format: `T{id} — {name} ({count})`, body 14px.
- Auto-refresh interval: 30 seconds (setInterval in useEffect, cleared on unmount).

### Threat Hunt View (DASH-03, DASH-04)
- Full-width search input at top, placeholder: "Search threats by description, IOC, or behaviour…".
- Submit on Enter key or "Search" button (accent background).
- Results rendered as ordered list below input, each item: similarity score badge (accent fill, label 12px/600, value 0.00–1.00) + IOC value (body 14px) + type label (label 12px/muted).
- Entire result row is clickable → opens OpenCTI deep-link in new tab.
- Cursor: pointer on result rows.
- No auto-refresh — results are static until new search.

### Briefings View (DASH-05, AIBR-03, AIBR-04)
- Period selector (24h / 72h / 7d) as `<select>` element + "Generate Briefing" button (accent) on same row at top.
- Generation state: button disabled + spinner + "Generating…" label while polling. Poll every 3 seconds via setInterval. Stop on status "done" or "error".
- Briefings list below, newest first. Each row: generated-at timestamp (label/12px/muted), period label (e.g. "24h"), status badge (text only: "done" / "generating" / "error"), PDF download link (anchor tag, "Download PDF", body/14px/accent) — link only shown when status == "done".
- On initial load: call `GET /briefings` to populate list.
- After generation completes: refresh list by calling `GET /briefings` again.

## States and Interactions

### Loading States
- Overview cards: show skeleton placeholder (grey rectangle, same card dimensions, opacity 0.3) while fetch is in flight.
- Threat Hunt: "Searching…" text in body/14px/muted replaces results list during fetch.
- Briefings generation: spinner inline with "Generating…" text on button row. Button disabled (opacity 0.5, cursor not-allowed).

### Empty States
- Overview feed grid (no feeds returned): heading "No feeds configured" (heading/20px), body "Feed orchestrator returned no data. Check service health." (body/14px/muted).
- Threat Hunt (no results): heading "No results found" (heading/20px), body "Try a broader query or wait for more IOCs to be indexed." (body/14px/muted).
- Briefings list (no briefings yet): heading "No briefings yet" (heading/20px), body "Select a time period and click Generate Briefing to create your first summary." (body/14px/muted).

### Error States
- Fetch failure (any view): inline error message in destructive text (#ef4444), body/14px: "Could not load data — {service name} unreachable. Retry in {interval}s." No modal dialogs.
- Generation error (briefings): status badge shows "error" in destructive color; no auto-retry; user must click "Generate Briefing" again.

### Destructive Actions
None in this phase — no delete, no clear, no destructive mutations. Confirmation dialogs not needed.

## API-to-View Binding

| View | Endpoint | Port | Polling |
|------|----------|------|---------|
| Overview — feed health | GET /feeds/status | 8001 | 30s interval |
| Overview — IOC stats + techniques | GET /stats | 8003 | 30s interval |
| Threat Hunt — search | POST /search | 8002 | on submit only |
| Briefings — list | GET /briefings | 8003 | on load + after generation |
| Briefings — generate | POST /generate | 8003 | once, then poll |
| Briefings — poll status | GET /briefings/{id} | 8003 | 3s interval until done/error |
| Briefings — download | GET /briefings/{id}/pdf | 8003 | on click (anchor href) |

**Source:** All from CONTEXT.md decisions D-04 through D-11.

CORS: All three FastAPI services allow `http://localhost:3000` — wired in this phase per D-14.
