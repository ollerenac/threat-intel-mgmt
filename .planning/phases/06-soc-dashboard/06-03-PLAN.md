<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser renders IOC values from API | JSON values rendered as React text children — never as HTML |
| Browser opens opencti_url in new tab | External URL from semantic-engine response; used as anchor href |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-06-03-01 | Tampering | XSS via IOC values (result.value, feed names, technique names) | high | mitigate | React JSX renders all values as text nodes (not innerHTML). No dangerouslySetInnerHTML used anywhere — enforced by acceptance criterion and verify grep gate. |
| T-06-03-02 | Spoofing | Open redirect via opencti_url from semantic-engine results | low | accept | Demo scope; opencti_url is always "http://localhost:8080/..." in dev. rel="noopener noreferrer" prevents tab-napping. Future hardening: validate URL prefix matches http://localhost:8080 before setting href. |
| T-06-03-03 | Denial of Service | Threat Hunt query length unbounded toward backend | medium | mitigate | Client-side guard: query.length > 500 returns early before fetch. Mirrors semantic-engine server-side 400 response for same condition (T-04-04-03 from Phase 4). |
| T-06-03-04 | Denial of Service | Briefings generation polling continues if tab closed | low | mitigate | setInterval id is local to handleGenerate scope; cleared on done/error. React strict mode does not double-fire this pattern (unlike useEffect). Acceptable: user navigating away stops rendering but interval fires until done — acceptable for demo scope. |
</threat_model>
