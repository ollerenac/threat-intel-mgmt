  <action>
    Step 1 — Create services/dashboard/Dockerfile (multi-stage per D-12):
    Stage 1 (build): FROM node:20-slim AS build; WORKDIR /app; COPY package*.json ./; RUN npm ci; COPY . .; RUN npm run build
    Stage 2 (serve): FROM nginx:alpine; COPY --from=build /app/dist /usr/share/nginx/html; COPY nginx.conf /etc/nginx/conf.d/default.conf; EXPOSE 80
    No CMD needed — nginx:alpine has its own default CMD.

    Step 2 — Create services/dashboard/nginx.conf (SPA routing per RESEARCH.md Pattern 2):
    server block: listen 80; root /usr/share/nginx/html; index index.html;
    location / block: try_files $uri $uri/ /index.html;
    This enables SPA routing — any unknown path falls back to index.html, where React's useState tab routing takes over.

    Step 3 — Fix docker-compose.yml soc-dashboard service block (Edit tool, scoped replacement):
    Current stub (lines 358-376) has four bugs. Apply all fixes in one Edit:
      a) build path: ./services/soc-dashboard → ./services/dashboard
      b) port mapping: "3000:3000" → "3000:80"
      c) Remove all VITE_* environment variables — they have no runtime effect with static nginx serve (Pitfall 5 from RESEARCH.md). Remove the entire environment: block from soc-dashboard.
      d) depends_on: replace the current list (opencti, intel-extractor, semantic-engine, briefing-generator) with:
           feed-orchestrator:
             condition: service_healthy
           semantic-engine:
             condition: service_healthy
           briefing-generator:
             condition: service_healthy
      The dashboard does not call intel-extractor or opencti directly. feed-orchestrator must be healthy before dashboard starts (it provides /feeds/status).

    Step 4 — Fix docker-compose.yml feed-orchestrator service block:
    Find the feed-orchestrator service block using grep to locate its line number first.
    Add ports key (if missing): "127.0.0.1:8001:8001" — bind to localhost only (feed orchestrator is an internal service; the dashboard calls it from the browser at localhost:8001, so host binding is required)
    Add healthcheck block (urllib.request pattern — no curl in python:3.12-slim):
      test: ["CMD", "python", "-c", "import urllib.request; urllib.request.urlopen('http://localhost:8001/health')"]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 60s
    Use Edit with scoped replacement targeting the feed-orchestrator block only.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → nginx (port 3000) | Static asset delivery; nginx serves only pre-built dist/ |
| docker-compose depends_on | Startup ordering guard; soc-dashboard waits for healthy backends |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-06-02-01 | Information Disclosure | VITE_* env vars in compose | low | mitigate | Remove all VITE_* env vars from soc-dashboard compose block — they have no effect at runtime and leak internal URL topology to anyone inspecting the container environment (Pitfall 5) |
| T-06-02-02 | Tampering | nginx serves arbitrary files from dist/ | low | accept | nginx:alpine serves only the pre-built Vite dist/ directory. No dynamic content, no CGI, no directory listing. SPA fallback limited to index.html. |
| T-06-02-03 | Denial of Service | feed-orchestrator port 8001 bound to 0.0.0.0 | medium | mitigate | docker-compose ports entry uses "127.0.0.1:8001:8001" — binds to localhost only, not all interfaces. Matches semantic-engine and briefing-generator host-binding convention. |
</threat_model>
