    Create services/semantic-engine/indexer.py. Implement the following:

    Imports: asyncio, logging, time, Optional from typing, chromadb, urlparse from urllib.parse,
    ollama; then from config import CHROMADB_URL, OLLAMA_URL, OLLAMA_EMBED_MODEL, OPENCTI_BASE_URL,
    POLL_INTERVAL_SECONDS; from opencti_client import build_pycti_client, list_all_indicators,
    list_indicators_since.

    Module-level state (exported for main.py):
    index_state: dict = {"status": "starting", "indexed": 0, "total": 0}

    ChromaDB client (module-level singleton, parsed from CHROMADB_URL):
    Parse host/port from CHROMADB_URL using urlparse. Create chromadb.HttpClient(host=..., port=...).
    COLLECTION_NAME = "ioc_embeddings"

    Ollama client (module-level singleton):
    _ollama = ollama.Client(host=OLLAMA_URL)

    _RETRY_DELAYS = [30, 60, 120] (same as intel-extractor)

    get_collection(): calls _chroma.get_or_create_collection(name=COLLECTION_NAME,
    configuration={"hnsw": {"space": "cosine"}}). CRITICAL: must use configuration= key (not
    metadata= key) and value {"hnsw": {"space": "cosine"}} — this is the ChromaDB 1.x API.
    The default metric is L2 which produces poor text similarity results (RESEARCH Pitfall 2).

    WATERMARK_ID = "_watermark_"
    read_watermark(collection) -> Optional[str]: get by id, return last_indexed_at from metadata or None.
    write_watermark(collection, timestamp: str): upsert with WATERMARK_ID, dummy embeddings=[[0.0]*768],
    documents=["watermark"], metadatas=[{"last_indexed_at": timestamp}].

    build_embed_text(indicator: dict) -> str: implement D-01/D-02/D-03 logic.
    ioc_type from indicator.get("x_opencti_main_observable_type", "Unknown").
    value from indicator.get("name", "").
    description from indicator.get("description") or "" (treat falsy as empty).
    labels from [lbl["value"] for lbl in (indicator.get("objectLabel") or [])].
    label_str = " ".join(labels).
    If description is truthy: return f"{ioc_type}: {value} — {description} {label_str}".strip().
    Else: return f"{ioc_type}: {value} [{label_str}]".strip().
    The em dash (—) is U+2014. Copy from PATTERNS.md to avoid ASCII hyphen substitution.

    _embed_with_retry(text: str) -> Optional[list[float]]: wraps _ollama.embed() in try/except
    with _RETRY_DELAYS. On success returns response.embeddings[0] (list of 768 floats).
    Use response.embeddings (plural) — NOT response.embedding (singular) (RESEARCH Pitfall 4).
    On final failure after all retries: log warning, return None. Caller skips None vectors.

    _index_batch(collection, indicators: list[dict]) -> int: iterate indicators, call build_embed_text,
    call _embed_with_retry. Skip indicator if vector is None. Call collection.upsert with:
    ids=[indicator["id"]], embeddings=[vector], documents=[embed_text],
    metadatas=[{"ioc_type": indicator["x_opencti_main_observable_type"],
                "value": indicator["name"],
                "opencti_url": f"{OPENCTI_BASE_URL}/dashboard/observations/indicators/{indicator['id']}",
                "embedded_text": embed_text}].
    Return count of successfully indexed items. Never index the _watermark_ sentinel ID.

    async run_index_loop(): coroutine (async def) that runs forever:
    1. Initialize pycti client and ChromaDB collection.
    2. Read watermark. If None: fetch all via list_all_indicators (full index).
       If present: fetch incremental via list_indicators_since(client, watermark).
    3. Update index_state["status"] = "indexing", index_state["total"] = len(indicators).
    4. Call _index_batch. Update index_state["indexed"] with running count.
    5. Write watermark with current UTC timestamp (datetime.utcnow().isoformat() + "Z").
    6. Update index_state["status"] = "ready".
    7. await asyncio.sleep(POLL_INTERVAL_SECONDS).
    8. Loop back to step 2 (incremental on subsequent cycles).
    Wrap the whole loop body in try/except Exception so a single cycle failure does not crash the loop.
    Log errors and set index_state["status"] = "error" on failure, then still sleep before retry.

    Note on ChromaDB volume mismatch (RESEARCH Pitfall 3): ChromaDB 1.x writes data to /data inside
    the container but docker-compose.yml mounts chromadata:/chroma/chroma. On restart, ChromaDB starts
    empty, watermark is also gone, so run_index_loop() runs a full re-index. This is expected behavior —
    do NOT attempt to fix the volume mount in this file. The full re-index is correct and safe.
