    Imports: logging, ollama; from config import OLLAMA_URL, OLLAMA_EMBED_MODEL.

    Module-level Ollama singleton: _ollama = ollama.Client(host=OLLAMA_URL).

    embed_query(text: str, ollama_client=None) -> list[float]: uses ollama_client if provided
    (for testability) else _ollama. Calls client.embed(model=OLLAMA_EMBED_MODEL, input=text).
    Returns response.embeddings[0]. Use embeddings (plural) — NOT embeddings() method and NOT
    .embedding (singular attribute) (RESEARCH Pitfall 4 — causes AttributeError at runtime).

    search(collection, query: str, ollama_client=None, n_results: int = 10,
           threshold: float = 0.3) -> list[dict]:
    1. Call embed_query(query, ollama_client).
    2. Call collection.query(query_embeddings=[query_vec], n_results=n_results,
       include=["distances", "metadatas", "documents"]).
    3. Iterate zip(results["distances"][0], results["metadatas"][0]).
    4. For each (dist, meta): score = round(1.0 - dist, 4).
       CRITICAL: ChromaDB returns cosine DISTANCE (0=identical, higher=more different). Similarity
       is 1 - distance. Do NOT use raw distance as score. Do NOT filter on `dist > threshold`
       — filter on `score < threshold` (RESEARCH Pitfall 1, HIGH IMPACT).
    5. If score < threshold: continue (D-07).
    6. Append dict with keys: ioc_type (from meta["ioc_type"]), value (from meta["value"]),
       score (the converted similarity float), opencti_url (from meta["opencti_url"]),
       embedded_text (from meta["embedded_text"]) (D-08).
    7. Return output list. ChromaDB already returns results ordered by distance ascending
       (most similar first), so no additional sort needed.

    The function signature accepts ollama_client as optional second positional arg so tests can
    inject mock_ollama directly without monkeypatching the module-level singleton.
  </action>

    Imports: asyncio, logging, contextlib.asynccontextmanager, fastapi.FastAPI,
    fastapi.HTTPException; import indexer; import searcher; from config import SIMILARITY_THRESHOLD.

    logging.basicConfig(level=logging.INFO).

    Lifespan context manager (use asynccontextmanager, not @app.on_event which is deprecated):
    @asynccontextmanager async def lifespan(app: FastAPI):
        asyncio.create_task(indexer.run_index_loop())  # D-05: fire-and-forget, does not block
        yield
    This is the correct pattern for FastAPI 0.115 (intel-extractor uses BackgroundTasks on
    per-request basis; semantic-engine needs a startup-triggered persistent loop — hence lifespan).

    app = FastAPI(title="semantic-engine", version="1.0.0", lifespan=lifespan)

    @app.get("/health"): returns dict spreading indexer.index_state plus status="ok".
    Response shape: {"status": "ok", "index_status": "starting"|"indexing"|"ready"|"error",
    "indexed": int, "total": int}. This endpoint MUST return immediately (D-05) — never await
    or block on indexer state. The spread of index_state keys is safe because it is a simple dict.

    @app.get("/search"): accepts q: str parameter and n_results: int = 10 (D-06 default).
    V5 input validation (RESEARCH.md Security Domain):
    - If not q: raise HTTPException(status_code=400, detail="q parameter required")
    - If len(q) > 500: raise HTTPException(status_code=400, detail="q too long (max 500 chars)")
    Call searcher.search(indexer.get_collection(), q, n_results=n_results, threshold=SIMILARITY_THRESHOLD).
    Return {"query": q, "results": results, "count": len(results)}.

    No /jobs endpoint needed — semantic-engine is synchronous request/response for search,
    and indexing progress is exposed through /health (not a job system).
