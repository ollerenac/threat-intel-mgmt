<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Auth-Key header → abuse.ch API | API keys sent as HTTP headers; must come only from env var, never hardcoded |
| ThreatFox/MalwareBazaar JSON response → STIX pattern | Untrusted feed content is interpolated into STIX pattern strings |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-05-01 | Information Disclosure | Auth-Key header in request — leakage via exception logs | high | mitigate | Exception messages from requests (e.g., connection errors) must not include the Auth-Key value. Auth-Key is in the headers dict, not the URL; requests exception messages do not include headers by default. Verify no exception handler re-logs the headers dict. |
| T-02-05-02 | Tampering | STIX pattern injection — sha256_hash and ioc values from feeds | high | mitigate | Escape single quotes in all ioc values: value.replace("'", "\\'"). Applied in normalize() for both feeds before pattern string interpolation. |
| T-02-05-03 | Tampering | ThreatFox ioc_type=ip:port port not sanitized | medium | mitigate | Extract IP with rsplit(":", 1)[0] — the port is discarded and never reaches the STIX pattern. |
| T-02-05-04 | Denial of Service | ThreatFox get_iocs days=7 could return very large dataset | medium | mitigate | requests timeout=30 on the HTTP call. The data list is processed row by row; no in-memory accumulation. If row count exceeds a threshold (e.g., >50000), log a warning but continue processing. |
</threat_model>
