    2. Navigate to: Observations > Indicators (left sidebar under "Analysis" or "Observations").
    
    3. Confirm IOCs are visible:
       - There should be indicators from URLhaus (pattern contains url:value) and Feodo (pattern contains ipv4-addr:value)
       - Each indicator should show a "Confidence" value in the 15-100 range
       - Click one indicator and confirm the external_references show "URLhaus" or "Feodo Tracker" as source_name
    
    4. Check deduplication: Look at the total indicator count. Stop and restart the feed-orchestrator:
       `docker compose --profile feeds restart feed-orchestrator`
       Wait 3 minutes for initial runs to complete, then check the indicator count again in OpenCTI. The count should NOT double — pycti update=True merges duplicates.
    
    5. Check scheduled runs: Wait for one URLhaus cycle (1 hour) or check the logs:
       `docker compose logs --tail=200 --follow feed-orchestrator`
       After the URLhaus interval (1h) the log should show "[urlhaus] initial run starting" from the scheduler — this confirms FEED-06 (automatic scheduling).
       (You do NOT need to wait the full hour to approve — the scheduler job registration can be confirmed by log line "[APScheduler] Added job 'feed_urlhaus' to job store 'default'" at startup.)
    
    6. Verify confidence scores are non-zero:
       In the indicator list, sort by confidence descending. Feodo indicators should score higher (quality_weight=30) than URLhaus (quality_weight=15) for equally-recent IOCs.
  </how-to-verify>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| feed-orchestrator container → external feed URLs | URLhaus and Feodo are HTTPS; MalwareBazaar/ThreatFox/OTX are HTTPS with Auth-Key |
| feed-orchestrator container → OpenCTI API | Internal Docker network; bearer token auth |
| feed-orchestrator container → Redis | Internal Docker network; no auth; tim: namespace isolation |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-08-01 | Information Disclosure | docker compose logs reveals feed run details | low | accept | Logs show IOC counts and feed names but not API key values (per config.py security constraint). Demo environment; logs not exposed externally. |
| T-02-08-02 | Tampering | Restarting feed-orchestrator creates duplicate indicators | medium | mitigate | pycti update=True handles server-side merge by STIX pattern. Redis tim:ioc_seen: keys have 24h TTL; container restarts within 24h will use Redis to skip duplicate pycti calls. Integration step 4 verifies this. |
| T-02-08-03 | Denial of Service | All 5 feeds running in parallel on docker compose restart | low | accept | BaseFeed runs sequentially in main.py startup loop (not in parallel). APScheduler starts AFTER initial runs complete. |
</threat_model>
