<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| env vars → config.py | API keys arrive via Docker environment; must never be logged as values |
| pycti client → OpenCTI API | Outbound GraphQL over internal Docker network; token in header |
| Redis client → Redis | Internal Docker network; no auth in dev; tim: namespace isolation |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-02-01 | Information Disclosure | config.py — env var logging | high | mitigate | Log only bool(OTX_API_KEY) etc., never the key value itself. error_msg in status.py truncated to 500 chars and must not contain exception tracebacks that include env var values. |
| T-02-02-02 | Tampering | Redis key namespace collision | medium | mitigate | All keys prefixed with tim:feed_status: or tim:ioc_seen: per D-01. OpenCTI uses unprefixed keys. Namespaces are mutually exclusive. |
| T-02-02-03 | Denial of Service | _fetch_with_retry sleep(120) blocks thread | low | accept | BaseFeed runs in APScheduler ThreadPoolExecutor thread; max_instances=1 prevents concurrent runs of the same feed. 120s maximum wait is within all feed cadences (shortest is 1h). |
| T-02-02-04 | Tampering | pycti version mismatch (7.x vs 6.4.11) | high | mitigate | requirements.txt pins pycti==6.4.11 exactly. No wildcard or >= constraint. Dockerfile uses pip install without --upgrade flag. |
| T-02-02-SC | Tampering | pip install in Dockerfile | high | mitigate | All packages in RESEARCH.md Package Legitimacy Audit verified and approved. pycti==6.4.11 is official OpenCTI client. |
</threat_model>
