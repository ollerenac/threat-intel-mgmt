    OTXFeed.fetch() -> list[dict]:
    - from OTXv2 import OTXv2
    - otx = OTXv2(OTX_API_KEY, server="https://otx.alienvault.com")
    - since = datetime.now(timezone.utc) - timedelta(hours=self.interval_hours + 1)
    - since_str = since.strftime("%Y-%m-%dT%H:%M:%S")
    - pulses = otx.getall(modified_since=since_str)
    - Flatten: for each pulse, iterate pulse.get("indicators", []), set indicator["_pulse_name"] = pulse.get("name", "")
    - Return flat list of all indicator dicts

    OTX_TYPE_MAP (module-level constant):
    - "IPv4":             ("[ipv4-addr:value = '{v}']", "IPv4-Addr")
    - "domain":           ("[domain-name:value = '{v}']", "Domain-Name")
    - "hostname":         ("[domain-name:value = '{v}']", "Domain-Name")
    - "URL":              ("[url:value = '{v}']", "Url")
    - "FileHash-MD5":     ("[file:hashes.MD5 = '{v}']", "StixFile")
    - "FileHash-SHA256":  ("[file:hashes.'SHA-256' = '{v}']", "StixFile")
    - "FileHash-SHA1":    ("[file:hashes.'SHA-1' = '{v}']", "StixFile")
    All other OTX types: skip (return None from _parse_indicator)

    OTXFeed._parse_indicator(ind: dict) -> dict | None:
    - ind_type = ind.get("type", "")
    - ind_value = ind.get("indicator", "").strip()
    - if ind_type not in OTX_TYPE_MAP or not ind_value: return None
    - pattern_tmpl, obs_type = OTX_TYPE_MAP[ind_type]
    - val_safe = ind_value.replace("'", "\\'")
    - pattern = pattern_tmpl.replace("{v}", val_safe)
    - return dict(name, pattern, observable_type, labels, source_name, valid_from)

    OTXFeed.normalize(raw) -> list[dict]:
    - For each indicator dict, call _parse_indicator()
    - Filter None results
    - Return list of valid dicts

    Class attributes:
    - name = "otx"
    - quality_weight = QUALITY_WEIGHTS["otx"]  (25)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| OTX_API_KEY env var → OTXv2 SDK | API key passed to OTXv2() constructor; must not be logged |
| OTX pulse indicator data → STIX pattern | Untrusted indicator values from OTX pulses are interpolated into STIX patterns |
| OTXv2 SDK → otx.alienvault.com | External HTTPS call; only outbound connection in the feed-orchestrator |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-02-06-01 | Denial of Service | OTXv2.getall() without modified_since pulling all history | high | mitigate | Always pass modified_since=interval_hours+1 hours ago. Never call otx.getall() without this argument. Enforced in fetch() implementation. |
| T-02-06-02 | Tampering | STIX pattern injection from OTX indicator values | high | mitigate | val_safe = ind_value.replace("'", "\\'") applied before pattern interpolation for every indicator type. |
| T-02-06-03 | Information Disclosure | OTX_API_KEY logged in exception trace | high | mitigate | OTX_API_KEY is passed to OTXv2() constructor, not stored on self. Exception messages from OTXv2 SDK do not include the API key. Log only "[OTX] disabled: OTX_API_KEY not configured" (presence check), never the key value. |
| T-02-06-04 | Tampering | OTXv2 SDK last release 2021 — API behavior may have changed | medium | mitigate | Assumption A1 from RESEARCH.md: OTX API is stable since 2021. If SDK raises unexpected errors, BaseFeed._fetch_with_retry() catches and retries, then marks status=error. Non-blocking. |
</threat_model>
