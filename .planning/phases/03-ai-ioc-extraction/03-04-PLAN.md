    Function chunk_text(text: str, max_chars: int = 6000, overlap_chars: int = 600) -> list[str]:
      If len(text) <= max_chars, return [text].
      Sliding window: start=0, end=start+max_chars, step=max_chars-overlap_chars.
      Append text[start:end], advance start by step until start >= len(text).

    Function build_stix_pattern(ioc_type: str, value: str) -> Optional[tuple[str, str]]:
      Escape value: v = value.replace("'", "\\'")
      Look up ioc_type in IOC_TYPE_TO_STIX; return None if not found.
      Return (pattern_template.replace("{v}", v), observable_type).

    Function _parse_fallback_text(text: str) -> dict:
      Regex parse TYPE:VALUE lines. type_map covers IP, DOMAIN, URL, MD5, SHA1, SHA256, HASH.
      Return {"iocs": [...], "techniques": [], "malware_families": [], "threat_actors": []}.

    Function call_llm(client: ollama.Client, model: str, text: str) -> dict:
      Primary: client.chat(model=model, messages=[system+user], format="json",
      options={"temperature": 0, "num_ctx": 8192}).
      Parse: json.loads(response.message.content) — attribute path per Assumption A2.
      Use .get("iocs", []) when accessing parsed result (not ["iocs"]).

      on fallback), parse with _parse_fallback_text, return result.
      If fallback also raises: log warning, return empty dict with all 4 keys as empty lists.

    Function run_extraction(job_id: str, mode: str, content: Optional[bytes],
    url: Optional[str]) -> None:
      This is a plain def (NOT async def) — FastAPI runs plain def background tasks

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| PDF bytes / URL content → LLM | Untrusted document content sent to local Ollama — prompt injection risk |
| LLM output → STIX objects | JSON response from local LLM interpreted as structured data |
| extractor.py → OpenCTI | IOC values written to OpenCTI via pycti |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-03-04-01 | Tampering | LLM output schema validation | medium | mitigate | Parse with json.loads in try/except; use .get() for all key access; D-03 fallback; unexpected keys ignored |
| T-03-04-02 | Tampering | Prompt injection via PDF/URL content | low | accept | LLM output validated against expected JSON schema; malicious prompt content produces at worst garbage IOCs that are validated via build_stix_pattern type lookup before write |
| T-03-04-03 | Denial of Service | Large documents consuming memory | low | accept | chunk_text processes sequentially; peak memory is one chunk at a time; 200-page PDF accepted risk for demo scope |
| T-03-04-04 | Information Disclosure | IOC values in logs | low | mitigate | Log IOC counts and types only — do not log individual IOC values at INFO level |
</threat_model>
