<task type="checkpoint:human-verify" gate="blocking">
  <name>Checkpoint: Integration validation — PDF + URL extraction, ATT&CK links, OpenCTI visibility</name>
  <what-built>
    Full intel-extractor service running at localhost:8001. All 5 source files implemented
    (config.py, parser.py, opencti_client.py, extractor.py, main.py). Docker healthcheck active.
    Unit test suite: 8/8 GREEN. Service deployed with --profile extract.
  </what-built>
  <how-to-verify>
    Step 1 — PDF extraction test (AIEX-01, AIEX-04):
      Download a small PDF threat report or use any multi-page PDF you have on hand.
      Submit it:
        curl -X POST http://localhost:8001/extract \
          -F "file=@/path/to/your/report.pdf"
      Note the job_id returned. Poll until complete (status changes from "queued"/"processing"):
        curl http://localhost:8001/jobs/{job_id}
      Expected: status="complete", iocs_extracted > 0

    Step 2 — Check IOCs in OpenCTI (AIEX-04):
      Open http://localhost:8080 → Observations → Indicators
      Verify at least one new indicator appears (check "Created" timestamp = now)
      Expected: indicator with STIX pattern matching an IOC from your PDF

    Step 3 — URL extraction test (AIEX-02):
      curl -X POST http://localhost:8001/extract \
        -F "url=https://www.cisa.gov/news-events/cybersecurity-advisories"
      Poll for completion. Expected: status="complete" (may have 0 IOCs if page has no IOC-like strings — that is acceptable)

    Step 4 — ATT&CK relationship check (AIEX-03):
      If your PDF mentioned any ATT&CK technique names (e.g. "phishing", "lateral movement",
      "ransomware"), open an extracted indicator in OpenCTI and look for relationship edges
      in the "Relationships" tab. Expected: at least one "indicates" edge to an attack-pattern.
      NOTE: If the PDF had no technique mentions, this test is N/A — skip to step 5.

    Step 5 — Report object verification (AIEX-04, Assumption A6):
      In OpenCTI → Analysis → Reports, find the report created for your PDF job.
      Click into it. Expected: "Observed data" or "Indicators" tab shows the extracted
      IOCs as object_refs. If the report has zero objects, note this as Issue A6 (see below).

    Step 6 — Long document test (AIEX-05):
      Submit a PDF with 10+ pages (any long document works).
      Poll for completion. Expected: iocs_extracted matches manual count of visible IOCs
      (no silent truncation). Check logs for chunk processing:
        docker compose logs intel-extractor --tail 100 | grep chunk

    Report any failures with the exact curl output and docker logs excerpt.
  </how-to-verify>
