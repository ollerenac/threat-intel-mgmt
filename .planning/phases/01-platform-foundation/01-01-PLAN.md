<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 3: Install nvidia-container-toolkit (requires sudo + systemd restart)</name>
  <what-built>
    The research phase confirmed nvidia-container-toolkit is NOT installed (nvidia-ctk not on PATH, Docker runtime is runc not nvidia). The ollama service in docker-compose.yml uses driver: nvidia GPU passthrough. Without the toolkit, docker compose --profile platform up -d fails on the ollama service and docker compose ps shows ollama as unhealthy/restarting — blocking PLAT-03.
  </what-built>
  <how-to-verify>
    Run these five commands IN ORDER (all require sudo):

    1. curl -fsSL https://nvidia.github.io/libnvidia-container/gpgkey | sudo gpg --dearmor -o /usr/share/keyrings/nvidia-container-toolkit-keyring.gpg

    2. curl -s -L https://nvidia.github.io/libnvidia-container/stable/deb/nvidia-container-toolkit.list | sed 's#deb https://#deb [signed-by=/usr/share/keyrings/nvidia-container-toolkit-keyring.gpg] https://#g' | sudo tee /etc/apt/sources.list.d/nvidia-container-toolkit.list

    3. sudo apt-get update && sudo apt-get install -y nvidia-container-toolkit

    4. sudo nvidia-ctk runtime configure --runtime=docker

    5. sudo systemctl restart docker

    Then verify: docker run --rm --gpus all ubuntu nvidia-smi

    Expected: nvidia-smi output showing RTX 3050 with CUDA 13.0. If you see the GPU table, the toolkit is working correctly.
  </how-to-verify>
  <resume-signal>
    Type "nvidia-ok" after docker run --rm --gpus all ubuntu nvidia-smi shows the GPU table, or describe any error you encountered.
  </resume-signal>
