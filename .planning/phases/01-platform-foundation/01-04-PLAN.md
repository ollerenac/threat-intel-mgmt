<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| internet → Docker daemon | Image pulls download from Docker Hub / docker.elastic.co / ghcr.io |
| internet → Ollama | Model pulls download from ollama.com registry |
| host browser → OpenCTI localhost:8080 | Admin credentials cross this boundary on login |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Information Disclosure | OPENCTI_ADMIN_TOKEN in .env visible to local user | accept | Local filesystem; single-user demo deployment; no multi-user auth in v1 scope |
| T-04-02 | Tampering | Docker image pulls from public registries | mitigate | Images use specific version tags (6.4.0, 8.15.0, 7.2-alpine, 3.13) not :latest where possible; ollama and minio use :latest (no version pinning available for these) |
| T-04-03 | Information Disclosure | Ollama model weights downloaded over internet | accept | Models are public; no PII or secrets involved in download |
| T-04-04 | Elevation of Privilege | OpenCTI admin password in .env accessible to local processes | accept | Single-user local deployment; password generated by setup-env.sh; not committed to git |
| T-04-SC | Tampering | npm/pip/cargo installs | accept | No package installs in this plan; Docker images and Ollama models only |
</threat_model>
