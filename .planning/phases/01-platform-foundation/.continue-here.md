# BLOCKING CONSTRAINTS — Read Before Anything Else

> These are not suggestions. Each constraint was discovered through actual failure.

- [ ] CONSTRAINT: curl -f healthcheck trap — OpenCTI, Ollama, and ChromaDB healthchecks used `curl -f` which fails on HTTP 401/non-2xx responses even when the service IS healthy. Fixed in commit 9f4936b. Do NOT use `curl -f` for healthchecks against services that return 401 when healthy. Use `curl -s ... | grep -q <expected_text>` instead.

**Do not proceed until all boxes are checked.**

## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| `curl -f` healthcheck | `curl -f` exits non-zero on HTTP 401/4xx even when service is healthy. OpenCTI /health returns `{"status":"unauthorized"}` (401) when running normally. | advisory | Always use response-body grep: `curl -s URL \| grep -q expected_text` for healthchecks where the service may return auth-required responses |
