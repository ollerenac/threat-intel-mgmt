  <action>
connector-mitre is the only platform-profile service without a healthcheck block. DEPL-04 requires all services to have healthchecks visible in docker compose ps. Without a healthcheck, docker compose ps shows no health status for connector-mitre.

Add the following healthcheck block to the connector-mitre service, immediately after the mem_limit: 512m line:

    healthcheck:
      test: ["CMD-SHELL", "pgrep -f 'python.*mitre' || exit 1"]
      interval: 30s
      timeout: 10s
      retries: 3
      start_period: 120s

The start_period is 120s because connector-mitre waits for OpenCTI to be healthy (via depends_on) before starting, and the initial import setup takes time. The pgrep pattern matches the connector-mitre Python process that runs indefinitely (CONNECTOR_RUN_AND_TERMINATE=false).

Edit pattern:
  old_string: "    mem_limit: 512m\n    depends_on:\n      opencti:"
  new_string:  "    mem_limit: 512m\n    healthcheck:\n      test: [\"CMD-SHELL\", \"pgrep -f 'python.*mitre' || exit 1\"]\n      interval: 30s\n      timeout: 10s\n      retries: 3\n      start_period: 120s\n    depends_on:\n      opencti:"

DO NOT change the depends_on block, networks, restart, or environment variables of connector-mitre.
  </action>
