## Critical Anti-Patterns (carry forward from Phase 1)

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| curl -f against 4xx-healthy endpoints | OpenCTI /health returns HTTP 401 when healthy; `-f`/`-sf` treats 4xx as failure causing infinite polling loops | advisory | Always use `curl -s \| grep -qE 'unauthorized\|ok'` pattern; test probe manually before writing healthcheck |
| Host-side probe of internal-only ports | Ollama port 11434 is not host-bound by design; host curl always fails | advisory | Probe internal services with `docker compose exec <svc> <cmd>` not host-side curl |
| Assuming image has curl/wget | opencti, ollama, chromadb images ship without curl/wget; probes fail silently at healthcheck eval | advisory | Run `docker compose exec <svc> which curl` before writing any healthcheck that uses curl |
| Omitting opencti/worker | OpenCTI 6.x requires a separate worker process; without it STIX bundles sit in RabbitMQ forever and GraphQL counts stay 0 | advisory | Any OpenCTI 6.x compose stack must include `opencti/worker` — verify it is present before starting |
